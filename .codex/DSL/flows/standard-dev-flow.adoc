== 流程定位

本文件定义标准开发流程的阶段边界与衔接关系。
具体执行步骤与产物细节由执行层实现，不属于 DSL 规范范围。

== 阶段定义

阶段顺序：

1. `PLANNING`：规划阶段
2. `IMPLEMENTATION`：开发阶段
3. `SYSTEM_TEST`：系统测试阶段
4. `DELIVERY_DECISION`：交付裁决阶段

== 阶段衔接门禁

- 进入 `IMPLEMENTATION` 前，必须有规划结论 `PLAN_READY`。
- 进入 `SYSTEM_TEST` 前，必须完成开发阶段必要产物。
- 进入 `DELIVERY_DECISION` 前，必须有测试证据与文档更新记录。
