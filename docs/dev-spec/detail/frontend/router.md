# 路由规范

## 1. 基本原则

- 路由负责页面入口和页面级组织，不承载具体业务实现。
- `App.vue` 只保留路由出口和全局壳，不把页面内容直接写死在根组件里。
- 页面跳转优先通过路由维护，避免在组件里手写大量条件渲染切换页面。

## 2. 路由拆分

- 按业务域拆分路由，不要把所有页面塞进一个大列表里。
- 公共布局页与业务页分层组织，布局负责壳，业务页负责内容。
- 登录页、首页、详情页、列表页等基础页面保持独立路由，便于权限和状态管理。

## 3. 布局使用

- 统一布局、侧边栏、顶部栏等能力放在布局组件中，通过路由嵌套组合。
- 页面组件只关注内容区域，不重复实现外层导航结构。
- 多页面复用的壳层逻辑，优先抽成布局组件或路由守卫处理。

## 4. 路由命名

- 路由 `name`、路径、页面组件名保持语义一致。
- 路由路径尽量短且稳定，避免和页面内部实现强绑定。
- 需要权限控制的页面，优先在路由层表达，而不是散落在各个页面里。

## 5. 跳转与参数

- 页面跳转参数优先使用路由参数或查询参数，不要依赖临时全局变量。
- 对必填参数在进入路由时做校验，避免页面加载后才暴露错误。
- 路由参数变化会影响页面数据时，要明确刷新策略，避免旧状态残留。

## 6. 维护要求

- 新增页面前先确认路由归属、入口位置和布局层级。
- 路由配置变更后同步检查菜单、面包屑、权限和页面入口。
- 删除路由时一并清理对应菜单、跳转引用和文档说明。
