# 主题规范

## 1. 目标

- 统一管理前端视觉主题，避免颜色、字号、间距、圆角、阴影等值散落在各个页面和组件中。
- 所有和主题相关的值都应通过统一的主题变量、设计 token 或公共样式常量获取。
- 主题层只描述视觉语义和可复用的基础值，不承载页面业务逻辑。

## 2. 主题范围

- 颜色：主色、辅色、背景色、文字色、边框色、状态色。
- 尺寸：字号、行高、图标尺寸、控件高度、容器宽高。
- 间距：外边距、内边距、栅格间距、组件间距。
- 形状：圆角、边框宽度、阴影层级。
- 动效：过渡时间、缓动曲线、交互反馈节奏。

## 3. 管理原则

- 同一语义只能有一个来源，不允许多个文件分别定义同名或近似含义的主题值。
- 组件和页面禁止直接硬编码主题值，必须优先引用主题变量。
- 新增主题值前先检查是否已有可复用变量，避免重复定义。
- 如果某个值只在一个页面使用，先判断是否真的属于主题层，避免过度抽象。

## 4. 命名要求

- 主题变量命名应表达语义，不直接暴露实现细节。
- 颜色建议按用途命名，而不是按颜色字面值命名。
- 尺寸、间距、圆角等建议按层级或用途命名，保持可读性和可扩展性。

## 5. 使用边界

- 主题层负责提供稳定的基础值，不负责具体页面的排版拼装。
- 页面级局部差异可以在组件样式层调整，但不能破坏主题体系的一致性。
- 临时调试值、一次性样式值不能进入主题层。

## 6. 维护要求

- 主题变更需要评估全局影响，尤其是按钮、表单、卡片、导航等基础组件。
- 删除主题变量前先确认没有引用。
- 如果主题体系发生调整，相关样式规范和组件规范要同步更新。
