# 样式规范

## 1. 基本原则

- 样式以可维护、可复用为主，避免在组件里堆大量无结构的临时样式。
- 保持视觉层次清晰，所有与主题有关的颜色、字号、尺寸、间距、圆角、阴影、边框等数值都必须统一收口到主题样式管理。
- 不把业务逻辑写进样式，不把样式写成难以复用的页面特例。

## 2. 主题样式管理

- 颜色体系、字体体系、间距体系、圆角体系、阴影体系、边框体系统一放在主题样式中管理。
- 组件和页面中禁止直接写散落的主题值，优先使用主题变量、设计 token 或统一的样式常量。
- 新增主题值前先检查是否已有可复用变量，避免重复定义同一语义的颜色和尺寸。
- 主题样式变更要同步影响全局一致性，不允许同一个语义在不同页面出现不同取值。
- 主题管理的具体要求见 [主题规范](./theme.md)。

## 3. 作用域与组织

- 优先使用组件内样式或明确命名的样式类，减少全局污染。
- 需要全局共享的主题变量和基础排版，统一沉淀到公共主题文件或主题入口。
- 页面级样式只服务当前页面，通用样式抽到组件或公共层。

## 4. 视觉一致性

- 同类控件的尺寸、内边距、圆角、阴影保持统一。
- 空态、加载态、错误态的视觉表达保持一致，不在每个页面各写一套。
- 图标、按钮、表单、列表等基础元素优先复用现有样式体系。

## 5. 响应式要求

- 布局需要同时考虑桌面和移动端，避免只适配单一屏幕宽度。
- 宽度、间距、字号、容器尺寸尽量使用相对布局和可伸缩方案。
- 新增页面先确认小屏下的折行、滚动和收纳方式。

## 6. 可维护性

- 样式命名要能表达语义，不使用难以理解的临时缩写。
- 避免过深的选择器嵌套，减少后续覆盖成本。
- 公共变量、主题色、间距体系要可追踪，不要散落在多个文件里。
